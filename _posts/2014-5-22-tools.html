Ideation

  • Mind Node

    This is an awesome way to think visually, especially when mapping out complicated relational scructures, like web app flows and user stories.

  • A sketchbook

    For any thinker this is an old standby, I have tons of em. I think there are three in my backpack right now. The best way to quickly capture and iterate through ideas, great for explaining things to people, letting them explain things to you, and more! I have a tendency to use notebooks as mousepads.

Organization

  • Trello

    If I could only recomend one tool or app to anyone it would be Trello. If you use this, it will change your life. No joke. This is an orgaisational tool for your collaborators, your projects, your goals, your life. Organize everything. Srsly bro.

  • Google Calendar / iCal

    Although Trello does have a calendar of sorts, it's more focused around due dates as opposed to making appointments with people, so it can be pretty helpful to have fluency with sheduling apps.

  • Dropbox

    Do you like your things? Now you can have your things. Everywhere. I know Dropbox is not the cheapest solution out there, but the workflow, stability and compatability are off the charts. I have a free account and have been working for a while to get extra space here and there. My account is up to 26GB (it starts at 2 :)

Collaboration

  • InVision

    I use invision at work, and when I do projects with other people where we need to keep track of ui changes overtime and collaborate on iterations.

  • Github

    We use github at work to collaborate on all our code. I use it to host this blog, and I also use it to host code samples from my YouTube channel DevTips. You should subscribe to that.

  • LayerVault

    Think: github for creative files. I really like that becuase my files are version-controlled, I can delete things freely, and If I liked an old idea I just go back in my versions and find it. As opposed to hiding a layer becuase I dont want to follow that idea, but I don't want to throw it away. It just gives me a lot more confidence, and smaller file sizes.

  • Gmail

    I use the web interface for gmail. I don't use the Apple Mail app on my computer, it's always problematic and you end up with tons of diskspace eaten up by attachments. Gmail has gotten so good (a pretty) that it's preferable to a native app. Compound those reasons with the fact that it is pretty tightly wound with google calendars, docs and hangouts ... well, you got a winner.

  • Google Hangouts

    A video chat application that is intertwined with your email, text-chat, and you can make YouTube videos directly with it. Did I mention the iPhone app is awesome? Such Google, very video chat, wow.

Graphic Design

  • Photoshop

    Yes, I use photoshop. No, I don't believe it's the only solution. But I also think that people who skip a graphic program and go straight to the browser everytime are a bit premature. Designing in code is not as flexible as photoshop, not yet. And you can tell by looking at them if the site was designed in the browser. I have opinions on the topic, but the funny thing is this article was created in the browser. But let's be honest: it shows, doesn't it?

  • Illustrator

    I like Illustrator. I'm kinda gnarly with a pen tool. I think I could drop old Illy, but since its part of the Adobe CC, why not? I use it here and there. Mostly for icons and junk, logos, whatever.

  • Xscope

    I think this is a great tool for anyone making websites. Especially engineery types who dont want to open photoshop when they get mockups from a designery person. With Xscope you can take quick measurements, set up guides that run across your screen, and sample colors like woah. It's really great. Whenever I show it to developers their eyes light up!

  • Keynote

    Keynote is awesome, and the new update is easier than ever to use. (Let me hear you say centered artboard! woot!) I use it all the time to comunicate ideas. If I want to tell my boss what I'm trying to accomplish, just make a three slide keynote. Easy! Businessy-types get all excited when you show them a slideshow, haha. I also use Keynotes awesome animation engine to make little movies that I include in my YouTube videos all the time. It's really super duper!

  • FontCase

    Before he made the graphics app Sketch, the guy from Boheniam Coding made FontCase. I really like it as a way to organize and search my fonts for a project. I love that you can specify a font library location, making it possible to put your font library in Dropbox, allowing you to take your fonts everywhere you go! [Edit: looks like they removed this app, oops!]

  • Loremify

    When you need some Lorem quickly this dasboard widget has your back.

Dev Environment

  • my phone/tablet/browser

    [insert impressive stats about mobile usage this quarter] See? Gotta be on it.

  • MAMP

    Great for when I'm working on a project that needs an apache server and a local database, like Wordpress. Super simple and powerful. Love it.

  • Terminal

    I'm just learning, and I'm still quite clumsy in Terminal, but It's super important to new workflows, so I'm forcing myself.

  • SASS

    The best way to write CSS. I love it.

  • CodeKit

    Sometimes I use codekit to compile my sass, and other things, but not all the time, and less and less as I get better in the terminal. It has the best livereload feature I've used.

  • Bourbon

    Bourbon is a super awesome and lightweight SASS Mixin library that does a ton of cool and handy stuff.

  • jQuery

    I've never written javascript without using jQuery in some way. It's essential for me.

  • Wordpress

    Wordpress is my CMS of choice for people who need a CMS. It's really great software, and its so ubiqutious that you can count on most clients to have some degree of familiarity with it.

  • Jekyll

    Jekyll is my new website CMS of choice. It is awesome for a lot of reasons. I made a video about it.

  • Tower

    I don't love tower, but it does what it does pretty well. It's a GUI for git, which I need at this point. I'll phase this out as I get better with Terminal, but I still need it now.

  • Chrome

    I like Chrome, but I prefer Safari. The problem is that the lastest version of Safari ruined the dev tools, so it's kind of unusable for me.

  • Live Reload

    I use livereload right now in a kinda hacky way. It's supposed to auto refresh your browser as you work, but I'm still working out the kinks in my workflow. Sometimes I don't use it as it can be more hasle then its worth...
